# v9.0 切札バランス調整レポート

## 概要

エクストラフレームの切札（全28種）について、2枚組み合わせのバランスチェックと個別性能の底上げを実施しました。

---

## 1. 致命的コンボの調査

### 検出: ♠3 ドレイン + ♢A 防壁追加（無限ループ）

ドレイン（キーカードなし・コストなし）で自分のキャラクターをライフに戻し2点ダメージ → 防壁追加でライフから新たな防壁を設置 → 再びドレインで戻す、というループが成立。

**問題点:**
- ドレインにキーカードがないためカウンター不能
- 手札を一切消費しないため自己持続する
- 1サイクルあたり自分のライフ消費は1枚、相手に2点ダメージ
- 約12サイクルで相手のライフを削りきれる

**対処 → 防壁追加の修正**（後述）

### その他のコンボ: 問題なし

ドレイン+防壁追加と同条件（キーカードなし＋自己持続＋繰り返しダメージ）のパターンを全切札で調査し、他に該当するコンボがないことを確認。

| チェック対象 | 結論 | 理由 |
|--|--|--|
| ドレイン + 蘇生 | 問題なし | ドレインはライフに戻す（墓地ではない）ので蘇生で回収不可 |
| ドレイン + 援軍 | 問題なし | 援軍のコストBLで自己持続不可 |
| ドレイン + 各種召喚 | 問題なし | 全て手札消費が必要 |
| 内乱+暴動 の反復 | 問題なし | 毎回コストB+♣キーカード消費 |
| 救済 の反復 | 問題なし | 毎回♡キーカード消費 |

## 2. 実施した変更

### 2.1 ♢A 防壁追加 — ループ修正 + 機能拡充

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| actActCond | 表切札に「防壁追加」がある | 表切札に「防壁追加」がある + **1ターン1回制限** |
| actEffect | 相手の防壁≧自分の防壁の場合のみ | **条件削除**（1回制限で十分） |

**新規アクション: クイック防壁設置**

| 項目 | 内容 |
|------|------|
| actType | 召喚 |
| actSpeed / actTime | 即時 / **クイック** |
| actCost | L |
| actActCond | 表切札に「防壁追加」がある |
| actEffect | 手札からカード1枚を防壁として裏向きかつチャージ状態で場に出す |

> 即時スピード採用理由: キーカードがないため「キーカード条件なしの通常クイック禁止」ガイドラインに抵触しないよう即時にしている。

### 2.2 ♠4 威圧 — アクション追加

**新規アクション: クイック威圧**

| 項目 | 内容 |
|------|------|
| actType | 魔法 |
| actSpeed / actTime | 通常 / **クイック** |
| actCost | L |
| actKey | **♠A〜10** |
| actActCond | 表切札に「威圧」がある |
| actEffect | サイズが10以下の兵士を全て墓地に移す |

> 追加理由: 威圧は他の切札（スラッシュ、誘惑等）がv9.0で2アクション構成になった中、1アクションのみで見劣りしていた。キーカード♠A〜10を持つためカウンター可能で、既存の威圧（メイン・カウンター不能・自傷あり）との使い分けが生まれる。

### 2.3 ♢5 喪失 — アクション追加

**新規アクション: 虚脱**

| 項目 | 内容 |
|------|------|
| actType | 魔法 |
| actSpeed / actTime | 通常 / **クイック** |
| actKey | **♢A〜K** |
| actTarget | ドライブ状態のキャラクター1体 |
| actActCond | 表切札に「喪失」がある |
| actEffect | 対象のキャラクターを墓地に移す |

> 追加理由: 喪失はパッシブ（全員チャージ禁止）のみでアクションが0だった。パッシブと連携し「チャージ不可→ドライブ状態が残る→虚脱で除去」の戦略が生まれる。対象を「キャラクター」としており、防壁もクイックで除去可能。

### 2.4 ♣9 要塞 — アクション追加

**新規アクション: 反撃**

| 項目 | 内容 |
|------|------|
| actType | 誘発魔法 |
| actSpeed / actTime | 即時 / クイック |
| actTriggerCond | 表切札に「要塞」があるかつ、自分の防壁がダメージ判定にてブロッカーとして墓地に移った時 |
| actEffect | 墓地に移った防壁のカードの数字に等しいダメージをアタッカーのオーナーに与える。Jokerの場合は0 |

> 追加理由: 要塞はパッシブ（♠キーダメージ無効）の対象が投擲・死の槍のみと極めて狭かった。防壁ブロック時の反撃ダメージを追加することで「要塞」の名にふさわしいテーマになり、防壁を並べる防御戦略が攻撃抑止力を持つ。

## 3. 調整しなかった切札とその理由

### ♢8 修繕 — 調整不要

v9.0にて大幅な強化を実施済み。従来は♢J〜Kに縛っていた条件を撤廃。ストラテジー環境では手札の積み込みが可能なため、初ターンから意識的に大きな装備兵を作成でき、実戦的に強力。

### ♣8 リセット — 調整不要

相手のコンボを崩す切り札として明確な役割がある。効果を両プレイヤーに適用することでバランスを保っている。v8.2ではクラブのキーカードが必要なメインアクションだったが、v9.0で即時-クイックに変更し利用率の向上を図っている。

### ♣10 悪あがき — 調整不要（むしろ要注意）

悪あがきはもともと強力な切札。どのアクションでもキーカードがあれば無効化でき、手札全捨てのデメリットも実戦では逆手に取られることが多かった。

BlackPokerの手札上限は7枚で、2枚消費のアクションが多いため1ターンに使えるアクション数は約3回。この状況で毎ターン確定でアクションを1つ無効化できる効果は、手札全捨てのコストを差し引いても非常に強力。相手が対処手段（投擲など）を通せなくなり、逆転困難な局面が多発した。

今回の全体的な切札底上げの理由の一つが、悪あがきとファントムの相対的な強さにある。他の切札をこれらに近づけることでバランスを取る方針。
