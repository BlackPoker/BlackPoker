# ドキュメントスタイルチェック

このワークフローは、プロジェクト内の `.rst` ファイルに対して、スタイルガイドが遵守されているかを検証・修正します。

## 実行手順
1. スキル `rst_style_guide` (c:\Users\black\git\github\BlackPoker\.agents\skills\rst_style_guide\SKILL.md) に定義されているルール（指示語「以下」を「次」にするルール、および例外）を確認します。
2. プロジェクト内の `.rst` ファイル（特に `source/` 配下）を対象に、 `grep_search` 等を用いて「以下」という文字列を含むファイルを検索します。
3. 検索された箇所を一行ずつ確認し、スキル `rst_style_guide` のルールに抵触している（指示語として「以下」が使われている）場合は、「次」に置き換えます。
4. 修正が完了したら、差分（diff）を表示してユーザーに報告します。
