# ドキュメントスタイルチェック

このワークフローは、プロジェクト内の `.rst` ファイルおよび `.yaml` ファイル（`act.yaml` など）に対して、スタイルガイドが遵守されているかを検証・修正します。

## 実行手順
1. スキル `rst_style_guide` (c:\Users\black\git\github\BlackPoker\.agents\skills\rst_style_guide\SKILL.md) に定義されているルール（指示語「以下」の制限、および半角数字への統一ルール）を確認します。
2. 対象ファイルを検索し、スタイル違反がないか確認します：
   - **指示語のチェック**: `grep_search` 等を用いて「以下」という文字列を含むファイルを検索し、指示語として使われている箇所を「次」に置き換えます。
   - **数字表記のチェック**: `grep_search` 等を用いて、全角数字（０-９）を含むファイルを検索し、原則として半角数字に置き換えます。
   - **更新バージョンのカンマ区切りチェック**: `grep_search` 等を用いて、YAMLファイル内の `update:` の記述において、カンマの直後にスペースがない箇所がないか確認し、カンマの後に半角スペースを入れるように置き換えます。
3. 修正が完了したら、差分（diff）を表示してユーザーに報告します。
