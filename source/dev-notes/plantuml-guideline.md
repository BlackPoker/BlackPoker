# BlackPoker PlantUML 記述ガイドライン (PDF文字化け対策)

PDF生成時にPlantUML図内の日本語が文字化けする問題を防ぐためのガイドラインです。

## 禁止事項 (PDF出力時の制約)

以下の装飾や文字は、日本語（CJK文字）と組み合わせるとPDFレンダリング時に文字化けを引き起こすことが確認されています。**図内のノード（`:...;` や `while (...)`）では使用しないでください。**

1.  **太字装飾 (`**...**`)**
    *   ❌ `:カードを **墓地** に置く;`
    *   ⭕ `:カードを墓地に置く;` (強調が必要な場合は `【墓地】` などの記号を使用)
2.  **ノード内での改行コード (`\n`)**
    *   ❌ `:一行目\n二行目;`
    *   ⭕ PlantUMLの自動折り返し、または物理的な改行（ソースコード上での改行）を利用してください。
3.  **斜体装飾 (`//...//`)**
    *   (未検証ですが、太字と同様に化ける可能性が高いため避けてください)

## 推奨事項

*   **全角の括弧類 (`「`, `」`)**:
    *   最新の検証結果では、これら自体は文字化けの直接原因ではないことが分かりました。ただし、太字などと組み合わさると不安定になる可能性があるため注意が必要です。
*   **日本語フォントの明示設定 (`skinparam`)**:
    *   PDF出力時に漢字が化ける（特定の別の漢字に置き換わる）現象が発生した場合、以下の設定を図の冒頭（`@startuml` の直後など）に追加してください。
    *   `skinparam defaultFontName "Droid Sans Fallback"`
    *   (Docker環境の Alpine Linux ではこのフォントが標準的な日本語対応フォントとして利用可能です)
*   **名前なしの `@startuml`**:
    *   `@startuml diagram_name` とすると、稀にキャッシュやレンダリングで不具合が出ることがあるため、シンプルな `@startuml` を推奨します。
*   **シンプルなテキスト**:
    *   強調したい場合は `【 】` や `( )` 、あるいは `[ ]` などの半角記号で囲むようにしてください。

## 運用方法

新しい `.puml` ファイルを作成した際は、必ず `make latexpdf` (または GitHub Actions でのビルド) を実行し、生成されたPDFで文字化けがないか目視で確認してください。
